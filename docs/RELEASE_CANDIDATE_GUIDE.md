# 교사용 대시보드 1차 릴리즈 후보 안내

이 문서는 Phase 3-L 기준으로 검증된 기본 대시보드 1차 릴리즈 후보 범위를 정리한다. 실제 구현되어 확인된 기능만 사용 가능 기능으로 적고, 아직 구현되지 않은 기능은 미지원 또는 Stub 상태로 분리한다.

## 1. 현재 버전 개요

- 이 버전은 기본 대시보드 1차 릴리즈 후보이다.
- 학생명렬 CSV import 흐름은 브라우저에서 end-to-end로 검증되었다.
- 가져온 학생명렬은 기존 학생명렬 저장 구조에 저장되며 브라우저 새로고침 후에도 유지된다.
- 저장된 학생명렬은 학생명렬 위젯과 학생 기반 수업도구에서 사용된다.
- `.xlsx` 직접 import, 성적 파일 import, 상담 파일 import, mapping/alias 고도화는 아직 지원하지 않는다.

## 2. 현재 사용 가능한 주요 기능

현재 릴리즈 후보에서 실제 동작이 확인된 기능은 다음과 같다.

- 날짜 + 상세시간 표시
- 커스텀 알림 표시
- 포스트잇 표시
- 메모 표시
- task 기능 표시
- 학생명렬 위젯
- 학생명렬 CSV import
- 수업도구
  - 뽑기
  - 사다리
  - 룰렛
  - 타이머
  - 뽀모도로
  - QR코드

현재 릴리즈 후보는 기본 대시보드와 CSV 기반 학생명렬 흐름을 중심으로 본다. 아래의 미지원 또는 Stub 항목은 완료 기능으로 보지 않는다.

## 3. 학생명렬 CSV 형식

학생명렬 CSV는 다음 헤더를 기준으로 인식한다.

```csv
학년,반,번호,성명,비고
1,1,1,김가온,
1,1,2,이나래,
1,1,3,박도윤,
```

규칙:

- 필수 헤더는 `학년`, `반`, `번호`, `성명`, `비고`이다.
- 정상 인식된 학생만 저장된다.
- `번호` 또는 `성명`이 누락된 행은 오류 행으로 제외된다.
- 오류 행이 있어도 정상 행은 적용하고 저장할 수 있다.
- 적용 시 기존 저장 학생명렬은 선택한 CSV의 정상 학생 목록으로 교체된다.
- 현재 흐름은 중복 학생 병합, 반별 부분 병합, 되돌리기 히스토리를 제공하지 않는다.

## 4. 학생명렬 Import 사용 절차

1. 설정 화면으로 이동한다.
2. import 화면에 진입한다.
3. 학생명렬 CSV 파일을 선택한다.
4. 일반 CSV preview를 확인한다.
5. 학생 후보 미리보기를 확인한다.
6. 오류 행을 확인한다.
7. 정상 학생만 적용하고 저장한다.
8. 대시보드로 돌아가 학생명렬 위젯을 확인한다.
9. 브라우저를 새로고침한 뒤 저장된 학생명렬이 유지되는지 확인한다.
10. 뽑기, 사다리, 룰렛에서 저장된 학생 목록을 사용한다.

교사용 안내:

- 학생명렬 CSV는 `학년, 반, 번호, 성명, 비고` 헤더를 기준으로 인식한다.
- 적용하면 정상 인식된 학생만 저장되며, 오류 행은 제외된다.
- 저장된 학생명렬은 새로고침 후에도 유지되고 뽑기, 사다리, 룰렛에서 사용된다.
- 현재 `.xlsx` 직접 가져오기는 지원하지 않는다. 학생명렬표 양식을 CSV로 저장해 가져온다.

## 5. 학생 기반 수업도구 사용법

- 뽑기: 저장된 학생명렬을 대상으로 학생을 뽑을 수 있다.
- 사다리: 선택 반 학생을 참여자 목록으로 채울 수 있다.
- 룰렛: 선택 반 학생을 항목으로 채울 수 있다.

출석 상태가 반영되는 경우 현재 뽑기 기준은 다음과 같다.

- 제외: `absent`, `earlyLeave`, `officialAbsent`
- 포함: `late`, `unknown`

저장된 학생명렬이 없으면 학생 기반 수업도구는 기존 빈 상태 또는 기본 동작을 유지하며, 학생명렬을 먼저 가져오도록 안내한다.

## 6. 현재 미지원 또는 Stub 상태

다음 항목은 현재 릴리즈 후보의 완료 기능이 아니다.

- 시간표: 별도 실제 위젯 없음 또는 Stub 상태
- 학사일정: Stub 상태
- 바로가기: 별도 실제 위젯 없음
- 중요사이트 링크: 별도 실제 위젯 없음
- 날씨: 별도 실제 위젯 없음
- `.xlsx` 직접 import: 미지원
- 성적 파일 import: 미지원
- 상담 파일 import: 미지원
- mapping/alias 고도화: 미지원

위 항목은 후속 Phase 후보이며, 현재 릴리즈 후보에서 구현 완료로 안내하지 않는다.

## 7. 테스트 완료 범위

Phase 3-L 기준으로 다음 항목을 확인했다.

- `npm.cmd run build` 통과
- `git diff --check` 통과
- 브라우저 회귀 테스트 통과
- 학생명렬 CSV 선택 정상
- 일반 CSV preview 정상
- 학생 후보 미리보기 정상
- 정상 학생 저장 정상
- 오류 행 제외 정상
- 새로고침 후 저장 학생명렬 유지 정상
- 학생명렬 위젯에서 저장 학생 표시 정상
- 뽑기, 사다리, 룰렛에서 저장 학생명렬 사용 정상
- `.xlsx` 선택 시 unsupported 안내 유지
- 릴리즈 차단 문제 없음

## 8. 알려진 제한 사항

- 현재는 CSV 기반 학생명렬 import만 검증되었다.
- `.xlsx` 파일은 아직 직접 파싱하지 않는다.
- 성적 파일과 상담 파일은 아직 대시보드 데이터로 반영하지 않는다.
- 일부 기본 위젯 후보는 아직 실제 구현 전이거나 Stub 상태이다.
- 이 문서는 현재 구현 상태를 고정하기 위한 기준이며, 후속 Phase에서 변경될 수 있다.
